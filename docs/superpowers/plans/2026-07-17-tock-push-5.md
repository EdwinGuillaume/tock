# The 5 as a "push an opponent" card — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Change the **5** so it stops advancing your own marbles and instead moves one *opponent* marble forward exactly 5 on the ring (ring-only overshoot, protected targets excluded, may capture a third player), across engine, AI, and UI.

**Architecture:** A new `push` variant is added to the `Move` union. `moveSteps('5')` becomes `null` (the 5 joins `J`/`7` as a special card), and `getLegalMoves` grows a dedicated push branch that reuses the existing `resolveDestination` geometry with `enterLane: false` (guaranteeing ring-only). `applyMove` relocates the pushed opponent marble via the existing `relocate` (captures fall out for free). The bot gains one gated heuristic term, `opponentProgress`, active only on pushes. The UI reuses the pick-card → pick-marble flow, selecting an opponent marble.

**Tech Stack:** TypeScript (strict), Vitest, React + Ink, pnpm.
