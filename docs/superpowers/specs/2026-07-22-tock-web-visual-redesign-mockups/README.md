# Visual reference — Tock web redesign ("Feutrine & or")

These self-contained HTML mockups are the **validated visual source of truth**
for the redesign described in
[`../2026-07-22-tock-web-visual-redesign-design.md`](../2026-07-22-tock-web-visual-redesign-design.md).
They were reviewed and locked screen-by-screen during brainstorming. The
implementation must **match these** — they exist specifically to prevent drift.

They are **consolidated final versions**: where a later decision superseded an
earlier pass (menu presence model, board style, discard motion), only the final
state is kept here — there is no stale iteration to confuse.

## How to view

Each file is standalone (fonts via Google Fonts CDN; SVG boards generated by an
inline script). Open any file directly in a browser, e.g.
`open board.html`, or serve the folder with any static server. Best viewed at a
phone-ish width; several files animate on a loop to convey motion.

## What is authoritative vs illustrative

- **Authoritative**: the aesthetic (felt + gold, jewel marbles, Fredoka/Inter),
  the palette and tokens, the board rendering rules (B ring + C finish + rotated
  homes + carved socket ratios), each screen's layout and controls, and the
  motion character (echo not pulse, short/fast draw, reverse-draw discard, fade
  transitions with fast pass-cover).
- **Illustrative only** (do not treat as pixel-exact): the exact marble/socket
  placements used to populate a scene, home-pod sizing, and the phone-frame
  chrome (a mock device bezel, not part of the app). The real board geometry
  always comes from `@tock/core`'s `board2d`; these mockups reproduce it for
  ring 48 to look right, not to redefine it.
